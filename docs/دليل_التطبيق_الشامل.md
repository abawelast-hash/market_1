## 🎯 ما هو هذا التطبيق؟ (شرح مبسّط)

تخيّل أنك تريد شراء غسالة أو ثلاجة أو أثاث لبيتك في مدينة الرقة.  
عادةً ستضطر تروح من مستودع لمستودع وتسأل عن الأسعار وتتعب.

هذا التطبيق مثل **"دليل الهاتف" لكن للمتاجر والمستودعات**:
- تفتحه على هاتفك 📱
- تشوف كل المتاجر والمستودعات في الرقة 🏪
- تقارن الأسعار بين المتاجر 💰
- تحفظ المنتجات اللي عجبتك بقائمة أمنيات ❤️
- تشارك المنتجات على واتساب مع أهلك 📲
- **يشتغل بدون إنترنت!** 📡

## 🏗️ كيف يعمل التطبيق؟ (الآلية بالتفصيل)

### الفكرة البسيطة:

```
المستخدم (هاتف) ←→ الخادم (سيرفر) ←→ قاعدة البيانات (مخزن المعلومات)
```

### الشرح المفصّل:

#### 1️⃣ المستخدم يفتح التطبيق
- يكتب عنوان الموقع في المتصفح أو يضغط على أيقونة التطبيق
- التطبيق يُحمّل من الخادم (السيرفر)
- بعد أول مرة، التطبيق يتخزن على الهاتف ويشتغل بدون إنترنت

#### 2️⃣ التطبيق يطلب البيانات
- لما المستخدم يفتح صفحة المتاجر، التطبيق يرسل طلب للخادم: "أعطيني قائمة المتاجر"
- الخادم يروح لقاعدة البيانات يجيب المعلومات
- يرجعها للتطبيق بشكل منظم (JSON)

#### 3️⃣ التطبيق يعرض البيانات
- يأخذ البيانات ويحولها لبطاقات جميلة
- كل منتج ببطاقة فيها: الاسم، السعر، المتجر، صورة
- المستخدم يقدر يضغط ويشوف التفاصيل

#### 4️⃣ العمل بدون إنترنت
- التطبيق يحفظ نسخة من البيانات على هاتف المستخدم
- إذا انقطع الإنترنت، يعرض البيانات المحفوظة
- لما يرجع الإنترنت، يحدّث البيانات تلقائياً

## 🖥️ الأنظمة والتقنيات

### البنية التقنية (ما هي الأدوات المستخدمة)

```
┌─────────────────────────────────────────────┐
│                 المتصفح (Frontend)          │
│  HTML + CSS + JavaScript (Vanilla)          │
│  PWA + Service Worker + IndexedDB           │
├─────────────────────────────────────────────┤
│                 الخادم (Backend)             │
│  Node.js + Express.js                       │
├─────────────────────────────────────────────┤
│            قاعدة البيانات (Database)         │
│  SQLite عبر sql.js                          │
└─────────────────────────────────────────────┘
```

### شرح كل تقنية:

| التقنية | ما هي؟ | لماذا نستخدمها؟ |
|---------|--------|----------------|
| **Node.js** | برنامج يسمح بتشغيل JavaScript على الكمبيوتر (الخادم) | سريع وبسيط ومجاني |
| **Express.js** | إطار عمل لبناء واجهات API | يسهّل إنشاء مسارات الخادم |
| **sql.js** | قاعدة بيانات SQLite تعمل بـ JavaScript فقط | لا تحتاج تثبيت برامج إضافية |
| **HTML** | لغة بناء صفحات الويب | هيكل الصفحة (العظام) |
| **CSS** | لغة تنسيق وتجميل الصفحات | الألوان والأشكال (الجلد) |
| **JavaScript** | لغة البرمجة الرئيسية | التفاعل والحركة (العقل) |
| **PWA** | تطبيق ويب تقدمي | يتحول لتطبيق على الهاتف |
| **Service Worker** | برنامج يعمل في الخلفية | يخزّن الملفات للعمل أوفلاين |
| **IndexedDB** | قاعدة بيانات داخل المتصفح | تخزين محلي على جهاز المستخدم |
| **compression** | ضغط البيانات | تسريع التحميل على الإنترنت البطيء |
| **CORS** | حماية أمنية | السماح بالوصول من نطاقات مختلفة |
| **UUID** | معرّف فريد عالمي | لتعريف كل جهاز مستخدم |

## 🔄 مسار عمل المستخدم (User Flow)

### رحلة المستخدم الكاملة:

```
1. يفتح التطبيق
   ↓
2. يرى الصفحة الرئيسية (تصنيفات، منتجات مميزة، متاجر)
   ↓
3. يختار تصنيف (مثلاً: غرف نوم)
   ↓
4. تظهر قائمة المنتجات مع فلاتر (السعر، اللون، المادة)
   ↓
5. يضغط على منتج لرؤية التفاصيل
   ↓
6. يرى: الاسم، السعر، الوصف، المتجر، الضمان، المواصفات
   ↓
7. يختار:
   ├── ❤️ إضافة لقائمة الأمنيات
   ├── 📲 مشاركة على واتساب  
   ├── 📞 التواصل مع المتجر مباشرة
   └── 🏪 زيارة صفحة المتجر
```

### سيناريو واقعي:

> **أم أحمد** تريد شراء ثلاجة جديدة:
> 1. تفتح تطبيق "دليل الرقة" على هاتفها
> 2. تضغط على تصنيف "أجهزة كهربائية" ← "ثلاجات"
> 3. تشوف 5 ثلاجات من متاجر مختلفة
> 4. تقارن الأسعار (سامسونج 4 مليون، أريستون 3.5 مليون)
> 5. تحفظ اللي عجبتها بقائمة الأمنيات
> 6. تشارك صورة الثلاجة على واتساب لزوجها
> 7. تتواصل مع المتجر مباشرة عبر واتساب
> 8. كل هذا وهي في البيت! 🏠

| العمود | النوع | الوظيفة |
|--------|-------|---------|

## 🔌 كيف تتصل الأجزاء ببعضها؟

### رحلة طلب واحد:

```
📱 المستخدم يضغط "المتاجر"
    ↓
🌐 المتصفح يرسل طلب GET /api/stores
    ↓
🖥️ Express يستقبل الطلب (server.js)
    ↓
📂 يوجّهه لملف المسار (routes/stores.js)
    ↓
🗄️ يسأل قاعدة البيانات (database/schema.js)
    ↓
💾 SQLite يرجّع البيانات (dalil.db)
    ↓
📨 Express يرسل الرد بصيغة JSON
    ↓
📱 JavaScript يستقبل البيانات (api.js)
    ↓
🎨 يرسمها كبطاقات (ui.js + pages.js)
    ↓
💾 يحفظها للعمل أوفلاين (db.js → IndexedDB)
    ↓
👀 المستخدم يرى القائمة على الشاشة!
```

## ⚙️ الإعدادات المهمة

### المنفذ (Port)
- المنفذ الافتراضي: **3000**
- يمكن تغييره عبر متغير البيئة: `PORT=8080 node server.js`

### قاعدة البيانات
- المسار: `database/dalil.db`
- الحفظ التلقائي: كل **30 ثانية**
- الحفظ عند الإغلاق: **نعم** (عبر SIGINT/SIGTERM)

### التخزين المؤقت (Cache)
- الملفات الثابتة: **يوم واحد**
- صور WebP: **30 يوم**
- Service Worker: **لا يُخزّن مؤقتاً** (يتحدث دائماً)
- API في المتصفح: **30 دقيقة**

### العملة
- العملة الرئيسية: **ليرة سورية (SYP)**
- العملة الثانوية: **دولار أمريكي (USD)**
- تنسيق الأسعار: أرقام عربية مع فواصل

## 📝 ملاحظات مهمة

1. **لا تعدّل مجلد `node_modules`** - هذه المكتبات تُدار تلقائياً
2. **لا تحذف `dalil.db`** - هذا ملف قاعدة البيانات بكل المعلومات
3. **`sw.js` لازم يكون في مجلد `public`** - لأن Service Worker يحتاج يكون في نفس مستوى الملفات
4. **الأسعار بالليرة السورية** - التطبيق يتعامل بالأرقام الكبيرة (ملايين)
5. **واتساب هو قناة التواصل الرئيسية** - لأنه الأكثر استخداماً في سوريا
