## 🏁 التشغيل لأول مرة

### الخطوة 1: تنزيل المشروع
```bash
git clone https://github.com/abawelast-hash/market_1.git
cd market_1
```

### الخطوة 2: تثبيت المكتبات
```bash
npm install
```

### الخطوة 3: تعبئة قاعدة البيانات
```bash
npm run seed
```
> يُنشئ `database/dalil.db` ويملأه بالبيانات التجريبية

### الخطوة 4: تشغيل الخادم
```bash
npm start
```

### الخطوة 5: فتح التطبيق
- المتصفح: **http://localhost:3000**
- الهاتف (نفس الشبكة): **http://[IP-الكمبيوتر]:3000**

## 🐍 تشغيل مولّدات البيانات (اختياري)

### إعداد بيئة Python:
```bash
# إنشاء بيئة افتراضية
python -m venv .venv

# تفعيل البيئة (Windows)
.venv\Scripts\activate

# تفعيل البيئة (Mac/Linux)
source .venv/bin/activate

# تثبيت المكتبات
pip install faker pandas tqdm
```

### تشغيل المولّد V1 (1,200 متجر، 20,000 منتج):
```bash
python database/generate_simulation.py
# الخرج: database/dalil_simulation.db (~38 MB)
```

### تشغيل المولّد V2 (1,500 متجر، 25,000 منتج):
```bash
python database/generate_souq_v2.py
# الخرج: database/souq_v2.db (~57 MB)
# ⚠️ يستغرق عدة دقائق (بيانات ضخمة + فهارس)
```

### ملاحظات المولّدات:
- ملفات `.db` الكبيرة مُضافة في `.gitignore` (لن تُرفع لـ Git)
- المولّدات تُنشئ جداول خاصة بها (users, user_interactions, search_history)
- يمكن استخدام قواعد المحاكاة لاختبار ML/RecSys بدون التأثير على `dalil.db`

## 🗄️ إدارة قاعدة البيانات

### الموقع:
```
database/dalil.db              ← قاعدة التطبيق الأساسية
database/dalil_simulation.db   ← محاكاة V1 (~38 MB، gitignored)
database/souq_v2.db            ← محاكاة V2 (~57 MB، gitignored)
```

### إعادة ضبط البيانات:
```bash
# Windows
del database\dalil.db
npm run seed

# Mac/Linux
rm database/dalil.db
npm run seed
```

### الحفظ الاحتياطي:
```bash
# Windows
copy database\dalil.db database\dalil_backup.db

# Mac/Linux
cp database/dalil.db database/dalil_backup.db
```

### ملاحظات:
- قاعدة البيانات تُحفظ تلقائياً كل **30 ثانية**
- تُحفظ أيضاً عند إيقاف الخادم (Ctrl+C)
- **لا تحذف الملف أثناء عمل الخادم!**

## 🕵️ نظام الجاسوس (مراقبة الملفات)

### التشغيل:
```bash

```

### ماذا يفعل:
- يراقب كل ملفات المشروع (7 مجلدات)
- يسجل أي تغيير (إنشاء/تعديل/حذف) في `docs/سجل_التغييرات.md`
- يعرض إشعارات ملونة في Terminal
- **يتجاهل:** node_modules, .git, .db, .DS_Store, Thumbs.db, package-lock.json

### ملاحظة:
- الجاسوس يتجاهل التغييرات في مجلد `docs/` (لتجنب حلقة لا نهائية)
- يحفظ التغييرات بأسلوب debounce (كل 3 ثوانٍ)
- للإيقاف: **Ctrl+C**

## 🐛 حل المشاكل

### "Cannot find module 'express'"
```bash
npm install
```

### "EADDRINUSE: port 3000"
```bash
# Windows:
taskkill /f /im node.exe
# أو استخدم منفذ آخر:
set PORT=8080 && node server.js
```

### "الصفحة فارغة"
1. F12 → Console ← ابحث عن أخطاء
2. تأكد أن الخادم يعمل
3. تأكد أن `dalil.db` موجودة

### "التطبيق لا يعمل أوفلاين"
1. افتح التطبيق وأنت متصل بالإنترنت (أول مرة)
2. انتظر حتى يُخزّن الملفات
3. بعدها جرّب بدون إنترنت

### "خطأ في قاعدة البيانات"
```bash
del database\dalil.db
node database/seed.js
node server.js
```

### "مولّد Python لا يعمل"
```bash
# تأكد من تفعيل البيئة
.venv\Scripts\activate
# تأكد من المكتبات
pip install faker pandas tqdm
# شغّل المولّد
python database/generate_souq_v2.py
```

### "الجاسوس لا يسجل التغييرات"
```bash
# تأكد من تشغيله
npm run docs:watch
# تأكد أن ملف سجل_التغييرات.md موجود
```

## 📂 أين تجد كل شيء

| أريد أن... | الملف |
|-----------|-------|
| أعدّل ألوان التطبيق | `public/css/style.css` (ابحث عن `:root`) |
| أضيف متجر/منتج جديد | `database/seed.js` |

| أغيّر المنفذ | `server.js` (PORT) |
| أعدّل نص واتساب | `public/js/share.js` |
| أضيف API جديد | 1) ملف في `routes/` → 2) اربطه في `server.js` |
| أضيف صفحة جديدة | 1) دالة في `pages.js` → 2) مسار في `app.js` → 3) رابط في `index.html` |
| أولّد بيانات ضخمة | `python database/generate_souq_v2.py` |

## 🔄 تحديث التطبيق

### عند تعديل الكود:
1. أوقف الخادم (Ctrl+C)
2. عدّل الملفات
3. `npm start`

### عند تحديث Service Worker:
1. غيّر رقم الإصدار في `public/sw.js`
2. المتصفح سيكتشف التحديث تلقائياً

### عند إضافة ملف ثابت:
- أضفه في مصفوفة `STATIC_ASSETS` في `public/sw.js`
